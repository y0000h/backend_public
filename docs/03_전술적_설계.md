# 전술적 설계 (Tactical Design)

## 1. 개요
전략적 설계에서 정의한 컨텍스트 내부의 **코드 수준 설계 방법**을 다룸.

## 2. 주요 요소
| 개념 | 설명 |
|------|------|
| Entity | 고유 식별자를 가진 객체 |
| Value Object | 값으로 동일성 판단, 불변 권장 |
| Aggregate | 일관성을 유지하는 객체 묶음 |
| Aggregate Root | 외부 접근의 유일한 진입점 |
| Repository | 루트 단위 영속성 관리 |
| Domain Service | 특정 엔티티에 속하지 않는 비즈니스 규칙 |
| Application Service | 유스케이스 조립 및 트랜잭션 경계 관리 |

## 3. 예시 구조
```
Order (Aggregate Root)
 ├─ OrderItem
 └─ Delivery
```

## 4. 장단점
- ✅ 도메인 규칙 응집성 향상
- ✅ 테스트·변경 용이
- ⚠️ 설계 복잡도와 개발 비용 증가
