# 기존 계층형 아키텍처와의 차이점

## 1. 계층형 아키텍처 구조
```
Controller → Service → Repository
```

- 비즈니스 로직이 **Service 레이어에 흩어짐**
- 기술적 관심사 중심 설계

## 2. DDD 구조
```
Controller → Application Service → Domain Model → Repository
```

- 비즈니스 규칙이 **Domain 내부로 응집**
- 기술보다 **도메인 개념이 코드 구조를 주도**

## 3. 비교표
| 구분 | 계층형 | DDD |
|------|--------|-----|
| 중심 개념 | 기술 레이어 | 도메인 모델 |
| 로직 위치 | Service에 분산 | Entity/VO 내부 |
| 변경 영향 | 광범위 | 국소적 |
| 테스트 용이성 | 낮음 | 높음 |

## 4. 결론
계층형은 단순하지만 변경에 약하고,  
DDD는 복잡하지만 유지보수성과 확장성 면에서 유리하다.
